# コミットルール

## コミットの粒度

- コミットは **日本語一文で表現できる程度の細かい粒度** で行う
- 1つのコミットに複数の変更目的を混ぜない

## コミットメッセージ

- 日本語で、簡潔な一文にする
- 「何をしたか」が一文で伝わるようにする

## 粒度の例

### 良い例（細かく分割）

```
ルーティングにstep4〜6を追加
step4のビューテンプレートを作成
コントローラーにstep4アクションを追加
step4のバリデーションを実装
step4のテストを追加
```

### 悪い例（粒度が粗すぎる）

```
ステップフォーム（Step 4〜6）の実装
```
